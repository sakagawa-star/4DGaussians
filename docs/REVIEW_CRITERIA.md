# docs/REVIEW_CRITERIA.md

## レビュー基準
以下の観点でドキュメントおよびコードをレビューする。

### 1. ドキュメントレビュー

#### 1.1 曖昧性の排除
- 「適切に」「必要に応じて」「など」等の曖昧表現がないか
- すべての条件・閾値・パラメータが具体的数値で記述されているか

#### 1.2 エラーハンドリング・境界条件
- 各機能の異常系（入力不正、モデル読み込み失敗、GPU メモリ不足等）の振る舞いが定義されているか
- 境界値（0人検出、フレーム範囲外等）の扱いが明記されているか

#### 1.3 技術スタック・依存関係
- 使用ライブラリとバージョンが明記されているか
- 外部リソース（チェックポイント・データセットのURL、サブモジュールのリポジトリ、CUDA/PyTorchバージョン）の名前・パスが正確か

#### 1.4 データフロー・I/O定義
- モジュール間のデータ型・形式が明記されているか（BGR/RGB、テンソル型、shape等）
- ファイル入出力のパス規約・命名規則が定義されているか

#### 1.5 詳細設計の十分性
- 実装者が判断に迷う余地がないレベルまで詳細化されているか

### 2. コードレビュー

#### 2.1 重要度分類
発見した問題を以下の重要度で分類する:
- **高**: クラッシュ、データ破損、セキュリティ問題、メモリリーク
- **中**: 潜在的バグ、リソースリーク、未使用コード、型の不整合
- **低**: コードスタイル、命名、軽微な非効率

#### 2.2 修正対象
- 重要度:高と中を修正対象とする
- 重要度:低は報告のみ（修正は任意）
